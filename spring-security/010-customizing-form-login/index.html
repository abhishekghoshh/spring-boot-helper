    10. Customizing Form Login — URLs, Timeout, UI & Session Persistence

    10.1 Customizing Default Login/Logout URLs, Session Timeout & Form Login UI

    The Problem: Spring Security's defaults (auto-generated login page, /login URL, 30-minute timeout) are fine for prototyping but every production app needs custom branding, custom URLs, and configured session timeouts.

    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│  WHAT CAN BE CUSTOMIZED & WHERE                                                              │
├──────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│  ┌──────────────────────────┬─────────────────────────┬──────────────────────────────────┐  │
│  │  What                    │  Default Value           │  Where to Customize              │  │
│  ├──────────────────────────┼─────────────────────────┼──────────────────────────────────┤  │
│  │  Login Page URL (GET)    │  /login                  │  .formLogin(f -> f.loginPage())  │  │
│  │  Login Submit URL (POST) │  /login                  │  .loginProcessingUrl()           │  │
│  │  Login Success URL       │  / or saved request      │  .defaultSuccessUrl()            │  │
│  │  Login Failure URL       │  /login?error            │  .failureUrl()                   │  │
│  │  Logout URL (POST)       │  /logout                 │  .logout(l -> l.logoutUrl())     │  │
│  │  Logout Success URL      │  /login?logout           │  .logoutSuccessUrl()             │  │
│  │  Session Timeout         │  30 minutes              │  application.properties          │  │
│  │  Login Page UI           │  Auto-generated HTML     │  Custom Thymeleaf/JSP template   │  │
│  │  Form field: username    │  "username"              │  .usernameParameter()            │  │
│  │  Form field: password    │  "password"              │  .passwordParameter()            │  │
│  │  Remember Me             │  Disabled                │  .rememberMe()                   │  │
│  │  Session Cookie Name     │  JSESSIONID              │  application.properties          │  │
│  │  Max Sessions Per User   │  Unlimited               │  .maximumSessions()              │  │
│  └──────────────────────────┴─────────────────────────┴──────────────────────────────────┘  │

    10.1.1 Complete SecurityConfig — All Customizations in One Place
    @Configuration
@EnableWebSecurity
public class SecurityConfig {

    private final UserAuthEntityService userAuthEntityService;

    public SecurityConfig(UserAuthEntityService userAuthEntityService) {
        this.userAuthEntityService = userAuthEntityService;
    }

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            // ─── URL Authorization Rules ─────────────────────────────────────
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(
                    "/auth/**",           // Registration, public endpoints
                    "/custom-login",      // Custom login page (GET) must be permitAll!
                    "/css/**",            // Static CSS files
                    "/js/**",             // Static JS files
                    "/images/**",         // Static images
                    "/error"              // Error page
                ).permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .requestMatchers("/api/**").hasAnyRole("USER", "ADMIN")
                .anyRequest().authenticated()
            )

            // ─── Custom Form Login ───────────────────────────────────────────
            .formLogin(form -> form

                // ① Custom Login Page URL (GET request to show the login form)
                // DEFAULT: /login (auto-generated by DefaultLoginPageGeneratingFilter)
                // CUSTOM:  /custom-login → YOUR Thymeleaf/JSP template
                .loginPage("/custom-login")
                // IMPORTANT: When you set a custom loginPage():
                //   - DefaultLoginPageGeneratingFilter is NOT registered
                //   - DefaultLogoutPageGeneratingFilter is NOT registered
                //   - You MUST create a controller that handles GET /custom-login
                //   - You MUST add "/custom-login" to permitAll() or everyone gets 403!

                // ② Custom Login Processing URL (POST request that submits credentials)
                // DEFAULT: /login
                // This is the URL your HTML form's action attribute points to
                // UsernamePasswordAuthenticationFilter intercepts THIS URL
                .loginProcessingUrl("/perform-login")
                // Your HTML form: <form method="POST" action="/perform-login">

                // ③ Custom Form Field Names
                // DEFAULT: "username" and "password"
                // If your HTML form uses different input names:
                .usernameParameter("email")        // <input name="email" />
                .passwordParameter("passwd")       // <input name="passwd" />

                // ④ Success URL after login
                // DEFAULT: redirects to the originally requested URL (saved in requestCache)
                .defaultSuccessUrl("/dashboard", false)
                // false = redirect to saved request URL if it exists, else /dashboard
                // true  = ALWAYS redirect to /dashboard (ignore saved request)

                // ⑤ Failure URL when login fails
                // DEFAULT: /login?error
                .failureUrl("/custom-login?error=true")

                // ⑥ Custom Success Handler (advanced — overrides defaultSuccessUrl)
                // .successHandler((request, response, authentication) -> {
                //     // Custom logic: log login event, redirect based on role, etc.
                //     if (authentication.getAuthorities().stream()
                //             .anyMatch(a -> a.getAuthority().equals("ROLE_ADMIN"))) {
                //         response.sendRedirect("/admin/dashboard");
                //     } else {
                //         response.sendRedirect("/user/dashboard");
                //     }
                // })

                // ⑦ Custom Failure Handler (advanced — overrides failureUrl)
                // .failureHandler((request, response, exception) -> {
                //     String errorMessage = "Invalid credentials";
                //     if (exception instanceof LockedException) {
                //         errorMessage = "Account is locked";
                //     } else if (exception instanceof DisabledException) {
                //         errorMessage = "Account is disabled";
                //     }
                //     request.getSession().setAttribute("error", errorMessage);
                //     response.sendRedirect("/custom-login?error=true");
                // })

                // Allow everyone to access the login page and processing URL
                .permitAll()
            )

            // ─── Custom Logout ───────────────────────────────────────────────
            .logout(logout -> logout

                // ① Custom Logout URL (POST request to trigger logout)
                // DEFAULT: /logout
                .logoutUrl("/perform-logout")
                // Your HTML form: <form method="POST" action="/perform-logout">

                // ② Custom Logout Success URL
                // DEFAULT: /login?logout
                .logoutSuccessUrl("/custom-login?logout=true")

                // ③ Invalidate session on logout
                // DEFAULT: true
                .invalidateHttpSession(true)

                // ④ Delete cookies on logout
                .deleteCookies("JSESSIONID", "remember-me")

                // ⑤ Clear the SecurityContext
                // DEFAULT: true
                .clearAuthentication(true)

                // ⑥ Custom Logout Success Handler (advanced — overrides logoutSuccessUrl)
                // .logoutSuccessHandler((request, response, authentication) -> {
                //     // Log logout event, redirect to external page, etc.
                //     response.sendRedirect("/custom-login?logout=true");
                // })

                .permitAll()
            )

            // ─── Session Management ──────────────────────────────────────────
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED)
                .invalidSessionUrl("/custom-login?expired=true")
                .sessionFixation().migrateSession()      // Default: migrate session
                .maximumSessions(1)                       // Only 1 session per user
                .expiredUrl("/custom-login?expired=true") // Redirect when session expires
                .maxSessionsPreventsLogin(false)          // false = kick old session
            );

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12);
    }

    @Bean
    public AuthenticationProvider authenticationProvider() {
        DaoAuthenticationProvider provider = new DaoAuthenticationProvider();
        provider.setUserDetailsService(userAuthEntityService);
        provider.setPasswordEncoder(passwordEncoder());
        return provider;
    }
}
    10.1.2 Custom Login Page Controller
    @Controller
public class LoginController {

    // This handles GET /custom-login → shows the login form
    // Spring Security's UsernamePasswordAuthenticationFilter handles POST /perform-login
    @GetMapping("/custom-login")
    public String showLoginPage(
            @RequestParam(value = "error", required = false) String error,
            @RequestParam(value = "logout", required = false) String logout,
            @RequestParam(value = "expired", required = false) String expired,
            Model model) {

        if (error != null) {
            model.addAttribute("errorMessage", "Invalid email or password. Please try again.");
        }
        if (logout != null) {
            model.addAttribute("logoutMessage", "You have been logged out successfully.");
        }
        if (expired != null) {
            model.addAttribute("expiredMessage", "Your session has expired. Please login again.");
        }

        return "login";  // → templates/login.html (Thymeleaf)
    }
}
    10.1.3 Custom Login Page — Thymeleaf Template (templates/login.html)
    <!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Login - My Application</title>
    <link rel="stylesheet" th:href="@{/css/login.css}" />
</head>
<body>

<div class="login-container">

    <div class="login-card">
        <div class="logo">
            <img th:src="@{/images/logo.png}" alt="Logo" />
        </div>

        <h2>Welcome Back</h2>
        <p class="subtitle">Sign in to your account</p>

        <!-- ─── Error Messages ─────────────────────────────────────── -->
        <div th:if="${errorMessage}" class="alert alert-error">
            <span th:text="${errorMessage}">Error</span>
        </div>

        <div th:if="${logoutMessage}" class="alert alert-success">
            <span th:text="${logoutMessage}">Logged out</span>
        </div>

        <div th:if="${expiredMessage}" class="alert alert-warning">
            <span th:text="${expiredMessage}">Session expired</span>
        </div>

        <!-- ─── Login Form ─────────────────────────────────────────── -->
        <!--
            CRITICAL POINTS:
            1. action="/perform-login" must match .loginProcessingUrl("/perform-login")
            2. method="POST" is required (GET won't work)
            3. name="email" must match .usernameParameter("email")
            4. name="passwd" must match .passwordParameter("passwd")
            5. CSRF token is auto-injected by Thymeleaf (th:action)
        -->
        <form th:action="@{/perform-login}" method="POST">

            <div class="form-group">
                <label for="email">Email Address</label>
                <input type="email"
                       id="email"
                       name="email"
                       placeholder="john@example.com"
                       required
                       autofocus />
                <!-- name="email" matches .usernameParameter("email") in SecurityConfig -->
            </div>

            <div class="form-group">
                <label for="passwd">Password</label>
                <input type="password"
                       id="passwd"
                       name="passwd"
                       placeholder="Enter your password"
                       required />
                <!-- name="passwd" matches .passwordParameter("passwd") in SecurityConfig -->
            </div>

            <button type="submit" class="btn-login">Sign In</button>

            <!--
                The CSRF hidden field is AUTOMATICALLY added by Thymeleaf
                when you use th:action. It renders as:
                <input type="hidden" name="_csrf" value="uuid-token-here"/>
            -->
        </form>

        <div class="footer-links">
            <a href="/auth/register">Create an account</a>
            <a href="/auth/forgot-password">Forgot password?</a>
        </div>
    </div>

</div>

</body>
</html>

    10.1.4 Session Timeout Configuration (application.properties)
    # ─── Session Timeout ──────────────────────────────────────────────────────────
# DEFAULT: 30m (Tomcat default)
# This is INACTIVITY timeout — timer resets with every request
server.servlet.session.timeout=45m

# ─── Session Cookie Configuration ─────────────────────────────────────────────
server.servlet.session.cookie.name=JSESSIONID       # Cookie name (default)
server.servlet.session.cookie.http-only=true         # Prevent JavaScript access (XSS protection)
server.servlet.session.cookie.secure=true            # Only send over HTTPS
server.servlet.session.cookie.same-site=Lax          # CSRF protection (Lax or Strict)
server.servlet.session.cookie.max-age=-1             # -1 = session cookie (deleted when browser closes)
server.servlet.session.cookie.path=/                 # Cookie valid for entire app

# ─── Tomcat-Specific Session Settings ─────────────────────────────────────────
server.servlet.session.tracking-modes=cookie         # Only use cookies (not URL rewriting)
# URL rewriting appends ;jsessionid=ABC to URLs — SECURITY RISK! Always disable it.
    10.1.5 How Customization Maps to Internal Filters
    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│  HOW YOUR CUSTOM CONFIG MAPS TO SPRING SECURITY INTERNALS                                    │
├──────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│  Your Config                              │  Internal Effect                                │
│  ─────────────────────────────────────────┼──────────────────────────────────────────────── │
│                                            │                                                 │
│  .loginPage("/custom-login")              │  • LoginUrlAuthenticationEntryPoint:              │
│                                            │    entryPoint URL = "/custom-login"              │
│                                            │  • DefaultLoginPageGeneratingFilter:              │
│                                            │    NOT registered (you provide your own page)    │
│                                            │  • DefaultLogoutPageGeneratingFilter:             │
│                                            │    NOT registered                                │
│                                            │                                                 │
│  .loginProcessingUrl("/perform-login")    │  • UsernamePasswordAuthenticationFilter:          │
│                                            │    requestMatcher = POST /perform-login          │
│                                            │    (instead of default POST /login)              │
│                                            │                                                 │
│  .usernameParameter("email")              │  • UsernamePasswordAuthenticationFilter:          │
│                                            │    request.getParameter("email")                 │
│                                            │    (instead of "username")                       │
│                                            │                                                 │
│  .passwordParameter("passwd")             │  • UsernamePasswordAuthenticationFilter:          │
│                                            │    request.getParameter("passwd")                │
│                                            │    (instead of "password")                       │
│                                            │                                                 │
│  .defaultSuccessUrl("/dashboard", false)  │  • SavedRequestAwareAuthenticationSuccessHandler: │
│                                            │    defaultTargetUrl = "/dashboard"               │
│                                            │    alwaysUseDefaultTargetUrl = false             │
│                                            │                                                 │
│  .failureUrl("/custom-login?error=true")  │  • SimpleUrlAuthenticationFailureHandler:         │
│                                            │    defaultFailureUrl = "/custom-login?error=true"│
│                                            │                                                 │
│  .logoutUrl("/perform-logout")            │  • LogoutFilter:                                  │
│                                            │    requestMatcher = POST /perform-logout         │
│                                            │                                                 │
│  server.servlet.session.timeout=45m       │  • Tomcat's StandardManager:                      │
│                                            │    session.maxInactiveInterval = 2700 seconds    │
│                                            │                                                 │
└──────────────────────────────────────────────────────────────────────────────────────────────┘
