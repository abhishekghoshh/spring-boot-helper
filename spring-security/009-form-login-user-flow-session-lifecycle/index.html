    009 - Form Login User Flow & Session Lifecycle

    9. Form Login — Complete User Flow, Session Lifecycle & Internal Mechanics

    9.1 Form Login Defaults — Quick Reference

    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│  FORM LOGIN DEFAULTS IN SPRING SECURITY                                                      │
├──────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│  Default Login URL:           /login          (GET = show form, POST = submit credentials)  │
│  Default Logout URL:          /logout         (POST = invalidate session + clear cookie)    │
│  Default Login Page:          Auto-generated by DefaultLoginPageGeneratingFilter            │
│  Default Logout Page:         Auto-generated by DefaultLogoutPageGeneratingFilter           │
│  Default Session Cookie:      JSESSIONID      (name set by Servlet container)               │
│  Default Session Timeout:     30 minutes      (depends on Servlet container — Tomcat default)│
│  Default Auth Method:         Form Login IS the default when .formLogin() is configured     │
│  Default Success URL:         / (root) or the URL that triggered the login redirect         │
│  Default Failure URL:         /login?error                                                  │
│  Default Logout Success URL:  /login?logout                                                 │
│  CSRF Protection:             Enabled by default (required for form POST)                   │
│  Session Fixation:            Changes session ID after authentication (migrateSession)      │
│                                                                                              │
│  To configure these:                                                                         │
│  ┌────────────────────────────────────────────────────────────────────────────────────┐     │
│  │  server.servlet.session.timeout=30m    # in application.properties               │     │
│  │  server.servlet.session.cookie.name=JSESSIONID                                    │     │
│  │  server.servlet.session.cookie.http-only=true                                     │     │
│  │  server.servlet.session.cookie.secure=true     # only over HTTPS                 │     │
│  └────────────────────────────────────────────────────────────────────────────────────┘     │
│                                                                                              │
└──────────────────────────────────────────────────────────────────────────────────────────────┘

    9.2 Complete User Flow — Step by Step with All Internal Details

    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│  FORM LOGIN — COMPLETE USER FLOW (Every Single Step)                                        │
├──────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 1: User requests a protected resource (no JSESSIONID yet)                      ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  Browser → GET /dashboard                                                                   │
│            (No Cookie header — first visit, no session exists)                               │
│       │                                                                                      │
│       ▼                                                                                      │
│  Spring Security Filter Chain processes the request:                                         │
│                                                                                              │
│  ┌─ SecurityContextHolderFilter ─────────────────────────────────────────────────────┐      │
│  │  Looks for existing HttpSession → NONE found                                      │      │
│  │  SecurityContextHolder → empty (anonymous)                                        │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ AuthorizationFilter ─────────────────────────────────────────────────────────────┐      │
│  │  Rule: anyRequest().authenticated()                                                │      │
│  │  Current user: anonymous (not authenticated)                                       │      │
│  │  → ACCESS DENIED! Throws AccessDeniedException                                    │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ ExceptionTranslationFilter ──────────────────────────────────────────────────────┐      │
│  │  Catches AccessDeniedException for unauthenticated user                            │      │
│  │                                                                                    │      │
│  │  1. SAVES the original request URL in the session:                                │      │
│  │     requestCache.saveRequest(request)                                              │      │
│  │     → Saved: "GET /dashboard" (so we can redirect back after login)               │      │
│  │                                                                                    │      │
│  │  2. Delegates to AuthenticationEntryPoint:                                        │      │
│  │     LoginUrlAuthenticationEntryPoint → redirect to /login                         │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  Response: HTTP/1.1 302 Found                                                                │
│            Location: /login                                                                  │
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 2: Browser follows redirect — displays login page                              ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  Browser → GET /login                                                                       │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ DefaultLoginPageGeneratingFilter ────────────────────────────────────────────────┐      │
│  │  Intercepts GET /login and generates an HTML login form:                           │      │
│  │                                                                                    │      │
│  │  <html>                                                                            │      │
│  │  <body>                                                                            │      │
│  │    <h2>Please sign in</h2>                                                        │      │
│  │    <form method="POST" action="/login">                                           │      │
│  │      <input type="text" name="username" placeholder="Username"/>                  │      │
│  │      <input type="password" name="password" placeholder="Password"/>              │      │
│  │      <input type="hidden" name="_csrf" value="a1b2c3d4-e5f6-..."/>               │      │
│  │      <!--  ↑ CSRF token auto-injected by Spring Security -->                     │      │
│  │      <button type="submit">Sign In</button>                                      │      │
│  │    </form>                                                                        │      │
│  │  </body>                                                                           │      │
│  │  </html>                                                                           │      │
│  │                                                                                    │      │
│  │  NOTE: At this point Tomcat creates a TEMPORARY HttpSession to store the          │      │
│  │  CSRF token and the saved request (original URL "/dashboard").                    │      │
│  │  A preliminary JSESSIONID cookie may be set, but this session is NOT              │      │
│  │  yet authenticated — it only holds CSRF + saved request data.                     │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  Response: HTTP/1.1 200 OK                                                                   │
│            Content-Type: text/html                                                           │
│            Set-Cookie: JSESSIONID=TEMP_SESSION_1; Path=/; HttpOnly                          │
│            (HTML login form body)                                                            │
│                                                                                              │
│  Browser renders:                                                                            │
│  ┌──────────────────────────────────────┐                                                   │
│  │       Please sign in                 │                                                   │
│  │                                      │                                                   │
│  │  Username: [                    ]    │                                                   │
│  │  Password: [                    ]    │                                                   │
│  │                                      │                                                   │
│  │         [Sign In]                    │                                                   │
│  └──────────────────────────────────────┘                                                   │
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 3: User enters credentials and submits the form                                ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  User types: username = "john", password = "john123", clicks Sign In                        │
│                                                                                              │
│  Browser → POST /login                                                                      │
│            Content-Type: application/x-www-form-urlencoded                                  │
│            Cookie: JSESSIONID=TEMP_SESSION_1                                                │
│            Body: username=john&password=john123&_csrf=a1b2c3d4-e5f6-...                    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ CsrfFilter ─────────────────────────────────────────────────────────────────────┐      │
│  │  Validates that _csrf token in the form matches the one stored in session         │      │
│  │  → MATCH ✓ (prevents CSRF attacks)                                               │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ UsernamePasswordAuthenticationFilter ────────────────────────────────────────────┐      │
│  │                                                                                    │      │
│  │  This filter ONLY processes: POST /login                                          │      │
│  │                                                                                    │      │
│  │  1. Extract credentials from form data:                                           │      │
│  │     username = request.getParameter("username");  // "john"                       │      │
│  │     password = request.getParameter("password");  // "john123"                    │      │
│  │                                                                                    │      │
│  │  2. Create an UNAUTHENTICATED token:                                              │      │
│  │     UsernamePasswordAuthenticationToken authRequest =                              │      │
│  │         UsernamePasswordAuthenticationToken.unauthenticated("john", "john123");   │      │
│  │     // authenticated = false, authorities = empty                                 │      │
│  │                                                                                    │      │
│  │  3. Pass to AuthenticationManager:                                                │      │
│  │     Authentication result = authenticationManager.authenticate(authRequest);       │      │
│  │                                                                                    │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 4: Authentication (DB lookup + password verification)                          ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  ┌─ ProviderManager (AuthenticationManager) ─────────────────────────────────────────┐      │
│  │  Iterates through AuthenticationProviders                                          │      │
│  │  → DaoAuthenticationProvider handles username/password auth                       │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ DaoAuthenticationProvider ───────────────────────────────────────────────────────┐      │
│  │                                                                                    │      │
│  │  STEP 4a: Load user from database                                                 │      │
│  │  ─────────────────────────────────                                                 │      │
│  │  UserDetails user = userDetailsService.loadUserByUsername("john");                 │      │
│  │  → UserAuthEntityService → userAuthRepository.findByUsername("john")              │      │
│  │  → SQL: SELECT * FROM users WHERE username = 'john'                               │      │
│  │  → Returns: UserAuthEntity {                                                       │      │
│  │      id: 1, username: "john",                                                     │      │
│  │      password: "$2a$12$LJ3m4YXhQK0R1xLq3TP.hOFb7YCp3bVnGJ0i3IuZpZpx9uBfGHJKS",│      │
│  │      role: "ROLE_USER"                                                             │      │
│  │    }                                                                               │      │
│  │                                                                                    │      │
│  │  STEP 4b: Pre-authentication checks                                               │      │
│  │  ──────────────────────────────────                                                │      │
│  │  user.isAccountNonLocked()?    → true ✓                                          │      │
│  │  user.isEnabled()?             → true ✓                                          │      │
│  │  user.isAccountNonExpired()?   → true ✓                                          │      │
│  │                                                                                    │      │
│  │  STEP 4c: Password verification                                                   │      │
│  │  ──────────────────────────────                                                    │      │
│  │  passwordEncoder.matches("john123", "$2a$12$LJ3m4YXh...")                         │      │
│  │  → BCryptPasswordEncoder:                                                         │      │
│  │    - Extracts salt from stored hash                                               │      │
│  │    - Hashes "john123" with same salt                                              │      │
│  │    - Compares hashes → MATCH ✓                                                   │      │
│  │                                                                                    │      │
│  │  STEP 4d: Post-authentication checks                                              │      │
│  │  ───────────────────────────────────                                               │      │
│  │  user.isCredentialsNonExpired()? → true ✓                                        │      │
│  │                                                                                    │      │
│  │  STEP 4e: Create AUTHENTICATED token                                              │      │
│  │  ─────────────────────────────────────                                             │      │
│  │  return UsernamePasswordAuthenticationToken.authenticated(                         │      │
│  │      user,                     // principal = UserAuthEntity                       │      │
│  │      null,                     // credentials cleared for security                │      │
│  │      user.getAuthorities()     // [ROLE_USER]                                     │      │
│  │  );                                                                                │      │
│  │  // authenticated = TRUE                                                           │      │
│  │                                                                                    │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 5: Session creation — server saves auth state (THIS IS THE STATEFUL PART)     ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  Back in UsernamePasswordAuthenticationFilter → successfulAuthentication():                  │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ SecurityContextHolder ───────────────────────────────────────────────────────────┐      │
│  │  SecurityContext context = SecurityContextHolder.createEmptyContext();              │      │
│  │  context.setAuthentication(authenticatedToken);                                    │      │
│  │  SecurityContextHolder.setContext(context);                                        │      │
│  │                                                                                    │      │
│  │  SecurityContext now holds:                                                         │      │
│  │  ┌──────────────────────────────────────────────────────────┐                     │      │
│  │  │  Authentication: {                                        │                     │      │
│  │  │    principal: UserAuthEntity { username: "john" },        │                     │      │
│  │  │    credentials: null,                                     │                     │      │
│  │  │    authorities: [ROLE_USER],                               │                     │      │
│  │  │    authenticated: true                                     │                     │      │
│  │  │  }                                                        │                     │      │
│  │  └──────────────────────────────────────────────────────────┘                     │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ Session Fixation Protection ─────────────────────────────────────────────────────┐      │
│  │  SECURITY: Old session ID (TEMP_SESSION_1) is INVALIDATED                         │      │
│  │  A NEW session is created with a NEW ID to prevent session fixation attacks        │      │
│  │                                                                                    │      │
│  │  Old: JSESSIONID=TEMP_SESSION_1  → INVALIDATED (deleted)                         │      │
│  │  New: JSESSIONID=AUTH_SESSION_42 → CREATED (fresh session)                        │      │
│  │                                                                                    │      │
│  │  The saved request ("/dashboard") and CSRF token are migrated to the new session  │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ SecurityContextRepository (HttpSessionSecurityContextRepository) ────────────────┐      │
│  │                                                                                    │      │
│  │  SAVES the SecurityContext INTO the new HttpSession:                               │      │
│  │                                                                                    │      │
│  │  HttpSession session = request.getSession(true);  // new session                  │      │
│  │  session.setAttribute("SPRING_SECURITY_CONTEXT", securityContext);                │      │
│  │                                                                                    │      │
│  │  Tomcat's session store now has:                                                   │      │
│  │  ┌──────────────────────────────────────────────────────────────────────────┐     │      │
│  │  │  Session ID: "AUTH_SESSION_42"                                           │     │      │
│  │  │  ┌──────────────────────────────────────────────────────────────────┐   │     │      │
│  │  │  │  Attribute: "SPRING_SECURITY_CONTEXT"                            │   │     │      │
│  │  │  │  Value: SecurityContext {                                         │   │     │      │
│  │  │  │    authentication: {                                              │   │     │      │
│  │  │  │      principal: john,                                             │   │     │      │
│  │  │  │      authorities: [ROLE_USER],                                    │   │     │      │
│  │  │  │      authenticated: true                                          │   │     │      │
│  │  │  │    }                                                              │   │     │      │
│  │  │  │  }                                                                │   │     │      │
│  │  │  └──────────────────────────────────────────────────────────────────┘   │     │      │
│  │  │  creationTime: 2026-04-29T10:05:00                                      │     │      │
│  │  │  lastAccessedTime: 2026-04-29T10:05:00                                  │     │      │
│  │  │  maxInactiveInterval: 1800 seconds (30 minutes)                         │     │      │
│  │  └──────────────────────────────────────────────────────────────────────────┘     │      │
│  │                                                                                    │      │
│  │  THIS is what makes it STATEFUL:                                                  │      │
│  │  The server REMEMBERS the user's authentication in its memory.                    │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ AuthenticationSuccessHandler ────────────────────────────────────────────────────┐      │
│  │  1. Check requestCache → finds saved request: "GET /dashboard"                    │      │
│  │  2. Redirect to original URL the user wanted                                      │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  Response to browser:                                                                        │
│  ┌──────────────────────────────────────────────────────────────────────────────────┐       │
│  │  HTTP/1.1 302 Found                                                              │       │
│  │  Location: /dashboard                                                             │       │
│  │  Set-Cookie: JSESSIONID=AUTH_SESSION_42; Path=/; HttpOnly; Secure                │       │
│  │  ─────────────────────────────────────────────────────────────────                │       │
│  │                     ↑                                                             │       │
│  │                     This is the KEY cookie!                                       │       │
│  │                     Browser stores it and auto-sends with EVERY subsequent request│       │
│  │                     HttpOnly = JavaScript cannot access it (XSS protection)       │       │
│  │                     Secure = only sent over HTTPS (MITM protection)               │       │
│  └──────────────────────────────────────────────────────────────────────────────────┘       │
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 6: Subsequent requests — JSESSIONID only, NO username/password                 ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  Browser → GET /dashboard                                                                   │
│            Cookie: JSESSIONID=AUTH_SESSION_42   ← auto-sent by browser!                    │
│            (NO username, NO password in this request)                                       │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ SecurityContextHolderFilter ─────────────────────────────────────────────────────┐      │
│  │                                                                                    │      │
│  │  1. Read JSESSIONID from Cookie header: "AUTH_SESSION_42"                         │      │
│  │                                                                                    │      │
│  │  2. Look up HttpSession in Tomcat's session map:                                  │      │
│  │     SessionManager.findSession("AUTH_SESSION_42")                                 │      │
│  │     → FOUND! Session exists and has NOT expired.                                  │      │
│  │                                                                                    │      │
│  │  3. Extract SecurityContext from session:                                          │      │
│  │     SecurityContext ctx = (SecurityContext)                                        │      │
│  │         session.getAttribute("SPRING_SECURITY_CONTEXT");                          │      │
│  │     → Contains: Authentication { principal: john, authorities: [ROLE_USER] }      │      │
│  │                                                                                    │      │
│  │  4. Set in SecurityContextHolder (ThreadLocal):                                   │      │
│  │     SecurityContextHolder.setContext(ctx);                                        │      │
│  │     → Current thread now knows: user = john, role = ROLE_USER                     │      │
│  │                                                                                    │      │
│  │  5. Update session's last accessed time:                                          │      │
│  │     session.lastAccessedTime = now()                                              │      │
│  │     → Resets the 30-minute inactivity timeout                                     │      │
│  │                                                                                    │      │
│  │  ⚠️ NO PASSWORD CHECK! NO DATABASE QUERY!                                        │      │
│  │  Just a HashMap lookup by session ID — extremely fast.                            │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─ AuthorizationFilter ─────────────────────────────────────────────────────────────┐      │
│  │  Is john with ROLE_USER allowed to access /dashboard?                              │      │
│  │  Rule: anyRequest().authenticated() → john IS authenticated → ALLOW ✓            │      │
│  └────────────────────────────────────────────────────────────────────────────────────┘      │
│       │                                                                                      │
│       ▼                                                                                      │
│  Controller executes → returns Dashboard page                                                │
│                                                                                              │
│  Response: HTTP/1.1 200 OK                                                                   │
│            Content-Type: text/html                                                           │
│            (Dashboard HTML for john)                                                         │
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 7: More requests — same flow, session keeps alive                              ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  Browser → GET /api/orders                                                                  │
│            Cookie: JSESSIONID=AUTH_SESSION_42                                               │
│            → SecurityContextHolderFilter finds session → john authenticated ✓              │
│            → Controller returns orders → 200 OK                                             │
│                                                                                              │
│  Browser → GET /api/profile                                                                 │
│            Cookie: JSESSIONID=AUTH_SESSION_42                                               │
│            → SecurityContextHolderFilter finds session → john authenticated ✓              │
│            → Controller returns profile → 200 OK                                            │
│                                                                                              │
│  ... Every request: Cookie sent → Session found → Authenticated. No login needed!           │
│                                                                                              │
│                                                                                              │
│  ╔═══════════════════════════════════════════════════════════════════════════════════════╗   │
│  ║  STEP 8: What happens when JSESSIONID is missing or expired?                         ║   │
│  ╚═══════════════════════════════════════════════════════════════════════════════════════╝   │
│                                                                                              │
│  Scenario A: JSESSIONID cookie not present (new browser, cleared cookies)                   │
│  ─────────────────────────────────────────────────────────────────────                       │
│  Browser → GET /dashboard  (no Cookie header)                                               │
│       │                                                                                      │
│       ▼                                                                                      │
│  SecurityContextHolderFilter: No session cookie → empty SecurityContext                      │
│  AuthorizationFilter: Not authenticated → AccessDeniedException                              │
│  ExceptionTranslationFilter: → 302 Redirect to /login                                      │
│  → User must log in again!                                                                   │
│                                                                                              │
│  Scenario B: JSESSIONID exists but session expired (idle > 30 min)                          │
│  ─────────────────────────────────────────────────────────────────                            │
│  Browser → GET /dashboard                                                                   │
│            Cookie: JSESSIONID=AUTH_SESSION_42                                               │
│       │                                                                                      │
│       ▼                                                                                      │
│  Tomcat: SessionManager.findSession("AUTH_SESSION_42")                                      │
│  → Session found BUT lastAccessedTime + maxInactiveInterval < now()                         │
│  → Session EXPIRED → removed from memory → returns null                                     │
│  SecurityContextHolderFilter: No valid session → empty SecurityContext                       │
│  AuthorizationFilter: Not authenticated → AccessDeniedException                              │
│  ExceptionTranslationFilter: → 302 Redirect to /login                                      │
│  → User must log in again!                                                                   │
│                                                                                              │
│  Scenario C: JSESSIONID exists but is tampered/unknown                                      │
│  ─────────────────────────────────────────────────────                                       │
│  Browser → GET /dashboard                                                                   │
│            Cookie: JSESSIONID=FAKE_SESSION_999                                              │
│       │                                                                                      │
│       ▼                                                                                      │
│  Tomcat: SessionManager.findSession("FAKE_SESSION_999") → NOT FOUND                        │
│  → Same flow as above → 302 Redirect to /login                                             │
│                                                                                              │
└──────────────────────────────────────────────────────────────────────────────────────────────┘

    9.3 Complete Code Example — Form Login with Database Auth

    @Configuration
@EnableWebSecurity
public class SecurityConfig {

    private final UserAuthEntityService userAuthEntityService;

    public SecurityConfig(UserAuthEntityService userAuthEntityService) {
        this.userAuthEntityService = userAuthEntityService;
    }

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/auth/**").permitAll()   // Registration, public pages
                .requestMatchers("/css/**", "/js/**").permitAll()  // Static resources
                .anyRequest().authenticated()               // Everything else needs login
            )

            // ─── Form Login Configuration ────────────────────────────────────
            .formLogin(form -> form
                .loginPage("/login")                    // Custom login page URL (GET)
                .loginProcessingUrl("/login")           // Form action URL (POST)
                .defaultSuccessUrl("/dashboard", false) // Redirect after login
                // false = redirect to originally requested URL if available
                // true  = always redirect to /dashboard
                .failureUrl("/login?error=true")        // Redirect on failure
                .usernameParameter("username")          // HTML form field name (default)
                .passwordParameter("password")          // HTML form field name (default)
                .permitAll()                            // Allow everyone to see login page
            )

            // ─── Logout Configuration ────────────────────────────────────────
            .logout(logout -> logout
                .logoutUrl("/logout")                   // URL to trigger logout (POST)
                .logoutSuccessUrl("/login?logout=true") // Redirect after logout
                .invalidateHttpSession(true)            // Destroy session
                .deleteCookies("JSESSIONID")            // Remove session cookie
                .clearAuthentication(true)              // Clear SecurityContext
                .permitAll()
            )

            // ─── Session Management ──────────────────────────────────────────
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED) // Default
                .maximumSessions(1)                    // Only 1 session per user
                .expiredUrl("/login?expired=true")     // Redirect if session expires
            );

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12);
    }

    @Bean
    public AuthenticationProvider authenticationProvider() {
        DaoAuthenticationProvider provider = new DaoAuthenticationProvider();
        provider.setUserDetailsService(userAuthEntityService);
        provider.setPasswordEncoder(passwordEncoder());
        return provider;
    }
}

    9.5 Internal Filter Chain — Which Filter Does What

    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│  FILTERS INVOLVED IN FORM LOGIN (in order of execution)                                      │
├──────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│  Request arrives                                                                             │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 1. SecurityContextHolderFilter                                                      │    │
│  │    • Loads SecurityContext from HttpSession (via JSESSIONID cookie)                  │    │
│  │    • If session exists → user is already authenticated (no re-login)               │    │
│  │    • If no session → SecurityContext is empty (anonymous)                           │    │
│  │    • After request completes: saves SecurityContext back to session                 │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 2. CsrfFilter                                                                       │    │
│  │    • For POST requests: validates _csrf token from form matches session's token     │    │
│  │    • Prevents CSRF attacks (malicious sites can't forge valid forms)                │    │
│  │    • GET requests: generates and stores CSRF token in session                       │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 3. UsernamePasswordAuthenticationFilter                                              │    │
│  │    • ONLY activates for: POST /login                                                │    │
│  │    • Extracts username + password from form data                                    │    │
│  │    • Creates UsernamePasswordAuthenticationToken                                    │    │
│  │    • Calls AuthenticationManager.authenticate()                                      │    │
│  │    • On success: saves auth in SecurityContext, redirects to success URL            │    │
│  │    • On failure: redirects to /login?error                                          │    │
│  │    • For ALL OTHER requests: does nothing, passes to next filter                    │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 4. DefaultLoginPageGeneratingFilter                                                  │    │
│  │    • Intercepts GET /login (only if no custom login page is configured)             │    │
│  │    • Generates auto HTML login form with CSRF token                                 │    │
│  │    • If custom .loginPage("/login") with Thymeleaf → this filter is NOT registered │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 5. DefaultLogoutPageGeneratingFilter                                                 │    │
│  │    • Intercepts GET /logout                                                          │    │
│  │    • Generates auto HTML "Are you sure you want to log out?" page                   │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 6. LogoutFilter                                                                      │    │
│  │    • Activates for: POST /logout                                                    │    │
│  │    • Invalidates HttpSession                                                         │    │
│  │    • Clears SecurityContextHolder                                                    │    │
│  │    • Deletes JSESSIONID cookie (Set-Cookie: Max-Age=0)                              │    │
│  │    • Redirects to /login?logout                                                     │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 7. RequestCacheAwareFilter                                                           │    │
│  │    • After login success: retrieves the original URL from requestCache              │    │
│  │    • Replays the original request (e.g., redirect to /dashboard)                    │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 8. AuthorizationFilter                                                               │    │
│  │    • Checks if current user (from SecurityContext) has access to the URL            │    │
│  │    • Evaluates: .requestMatchers(...).hasRole(...) rules                            │    │
│  │    • Allowed → proceed to controller                                                │    │
│  │    • Denied → throws AccessDeniedException                                          │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐    │
│  │ 9. ExceptionTranslationFilter                                                        │    │
│  │    • Catches AccessDeniedException and AuthenticationException                      │    │
│  │    • If user not authenticated: saves original URL → redirect to /login             │    │
│  │    • If user authenticated but no permission: → 403 Forbidden                      │    │
│  └─────────────────────────────────────────────────────────────────────────────────────┘    │
│       │                                                                                      │
│       ▼                                                                                      │
│  Controller handles the request                                                              │
│                                                                                              │
└──────────────────────────────────────────────────────────────────────────────────────────────┘

    9.6 Session Timeout & Configuration

    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│  SESSION TIMEOUT — HOW IT WORKS                                                              │
├──────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│  Default: 30 minutes of INACTIVITY (not 30 minutes from login!)                             │
│                                                                                              │
│  How Tomcat calculates timeout:                                                              │
│  ┌──────────────────────────────────────────────────────────────────────────────┐           │
│  │  if (currentTime - session.lastAccessedTime > session.maxInactiveInterval)   │           │
│  │  → Session is EXPIRED → remove from memory                                   │           │
│  │                                                                               │           │
│  │  Every request that carries JSESSIONID updates lastAccessedTime.             │           │
│  │  So the 30-minute timer RESETS with every request.                            │           │
│  │                                                                               │           │
│  │  Example:                                                                     │           │
│  │  10:00 → Login (session created, lastAccessed = 10:00)                       │           │
│  │  10:15 → GET /orders (lastAccessed updated to 10:15)                         │           │
│  │  10:25 → GET /profile (lastAccessed updated to 10:25)                        │           │
│  │  10:40 → No request (idle)                                                    │           │
│  │  10:55 → Tomcat checks: 10:55 - 10:25 = 30 min → EXPIRED! Session removed.  │           │
│  │  10:56 → GET /orders → No session → Redirect to /login                      │           │
│  └──────────────────────────────────────────────────────────────────────────────┘           │
│                                                                                              │
│  Configuring timeout in application.properties:                                              │
│  ┌──────────────────────────────────────────────────────────────────────────────┐           │
│  │  # 30 minutes (default)                                                      │           │
│  │  server.servlet.session.timeout=30m                                           │           │
│  │                                                                               │           │
│  │  # 1 hour                                                                     │           │
│  │  server.servlet.session.timeout=60m                                           │           │
│  │                                                                               │           │
│  │  # 15 minutes                                                                 │           │
│  │  server.servlet.session.timeout=15m                                           │           │
│  │                                                                               │           │
│  │  # In seconds (900 seconds = 15 minutes)                                     │           │
│  │  server.servlet.session.timeout=900                                           │           │
│  │                                                                               │           │
│  │  NOTE: Tomcat's minimum is 1 minute. Values below 1m are rounded up to 1m.  │           │
│  └──────────────────────────────────────────────────────────────────────────────┘           │
│                                                                                              │
│  Configuring cookie properties:                                                              │
│  ┌──────────────────────────────────────────────────────────────────────────────┐           │
│  │  server.servlet.session.cookie.name=JSESSIONID     # Cookie name            │           │
│  │  server.servlet.session.cookie.http-only=true       # JS can't access       │           │
│  │  server.servlet.session.cookie.secure=true          # HTTPS only            │           │
│  │  server.servlet.session.cookie.same-site=Lax        # CSRF protection       │           │
│  │  server.servlet.session.cookie.max-age=-1           # -1 = browser session  │           │
│  │  # max-age=-1 means cookie is deleted when browser closes (session cookie)  │           │
│  │  # Set a positive value for "Remember Me" style persistent cookies          │           │
│  └──────────────────────────────────────────────────────────────────────────────┘           │
│                                                                                              │
│  Configuring session management in Java:                                                     │
│  ┌──────────────────────────────────────────────────────────────────────────────┐           │
│  │  .sessionManagement(session -> session                                       │           │
│  │      .sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED)  // default  │           │
│  │      .invalidSessionUrl("/login?invalid")           // invalid session ID   │           │
│  │      .sessionFixation().migrateSession()            // default: migrate     │           │
│  │      .maximumSessions(1)                             // 1 session per user  │           │
│  │      .maxSessionsPreventsLogin(false)                // kick old session    │           │
│  │      // false = new login kicks old session (default)                       │           │
│  │      // true  = new login is DENIED if session already exists               │           │
│  │  )                                                                           │           │
│  └──────────────────────────────────────────────────────────────────────────────┘           │
│                                                                                              │
└──────────────────────────────────────────────────────────────────────────────────────────────┘

    9.8 Key Takeaways — Form Login Flow Summary

    ┌──────────────────────────────────────────────────────────────────────────────────────────────┐
│  FORM LOGIN — KEY FACTS SUMMARY                                                              │
├──────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                              │
│  1. AUTHENTICATION HAPPENS ONCE:                                                            │
│     User provides username + password ONLY at POST /login.                                  │
│     After that, JSESSIONID cookie handles everything.                                       │
│                                                                                              │
│  2. SESSION = SERVER-SIDE STATE:                                                            │
│     Server stores SecurityContext (user info + roles) in HttpSession.                       │
│     Session lives in Tomcat's ConcurrentHashMap (JVM memory).                               │
│     This is WHY it's called STATEFUL authentication.                                        │
│                                                                                              │
│  3. JSESSIONID = OPAQUE POINTER:                                                            │
│     The cookie value is a random string — no user data inside.                              │
│     It's just a KEY to look up the session on the server.                                   │
│     Unlike JWT, it cannot be decoded or inspected by the client.                            │
│                                                                                              │
│  4. BROWSER AUTO-SENDS COOKIE:                                                              │
│     Set-Cookie: JSESSIONID=ABC → browser stores it.                                        │
│     Every subsequent request to the same domain → Cookie: JSESSIONID=ABC.                  │
│     Zero client-side JavaScript needed.                                                     │
│                                                                                              │
│  5. SESSION FIXATION PROTECTION:                                                            │
│     Spring Security generates a NEW session ID after successful login.                      │
│     Old session ID is invalidated. Prevents attackers from pre-setting a session.           │
│                                                                                              │
│  6. TIMEOUT = INACTIVITY BASED:                                                             │
│     Default: 30 minutes of NO requests.                                                     │
│     Every request resets the timer.                                                         │
│     After timeout → session deleted → user must log in again.                               │
│                                                                                              │
│  7. NO JSESSIONID = REDIRECT TO LOGIN:                                                      │
│     If the cookie is missing, expired, or invalid:                                          │
│     → SecurityContext is empty → not authenticated                                          │
│     → AuthorizationFilter denies access                                                     │
│     → ExceptionTranslationFilter redirects to /login                                       │
│                                                                                              │
│  8. IT IS THE DEFAULT IN SPRING SECURITY:                                                   │
│     When you add spring-boot-starter-security + .formLogin():                               │
│     → Auto-generates login/logout pages                                                     │
│     → Creates sessions automatically                                                        │
│     → SessionCreationPolicy.IF_REQUIRED (default)                                           │
│     → No JWT, no OAuth2, no extra config needed                                             │
│                                                                                              │
└──────────────────────────────────────────────────────────────────────────────────────────────┘
